# Kickoff

Frame the task through a quick **multiple-choice** interview, then start the work.
See `System/opening-move.md` for the framing.

1. **Get real context first.** Skim today's session log and the Kanban board
   [[01_Projects/To Do]] so the options you offer are actual in-flight work, not
   generic placeholders.
2. **Interview me with the `AskUserQuestion` tool — not prose.** In a single call,
   ask the four ingredients as multiple-choice questions, each with the
   recommended option first. Skip any I already gave in `$ARGUMENTS`.
   - **Outcome** — likely items from the board, plus "Other"
   - **Audience** — media / leadership / academic / industry / public (per [[voice]])
   - **Success** — a few concrete markers to choose from
   - **Mode** — think with me / draft / build / decide
3. **Read the stack & execute.** Pick the matching skill (skills-first); for
   writing, hold to [[voice]] + [[anti-style]]. State the smartest next move in one
   line, then proceed — confirm first only if it's large or irreversible.
