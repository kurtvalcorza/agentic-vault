# Skill Creator

Interactive guide for creating new agent skills that comply with organizational standards.

## What This Does

Walks you through the complete skill creation process using a 6-phase workflow:

1. **Discovery**: Understand the use case and identify triggers
2. **Planning**: Define scope, inputs, outputs, and edge cases
3. **Initialization**: Create directory structure with proper frontmatter
4. **Implementation**: Write SKILL.md with instructions and examples
5. **Testing**: Validate against standards and test execution
6. **Iteration**: Refine based on feedback and edge cases

## When to Use

- Creating a new skill from scratch
- Learning skill anatomy and best practices
- Ensuring compliance with Agent Skills Standard
- Understanding progressive disclosure patterns
- Documenting reusable workflows

## What You Learn

**Skill Anatomy**:
- Dual-documentation pattern (SKILL.md + README.md)
- Frontmatter requirements (name, description)
- Directory structure (skill-name/ with references/, templates/)
- Trigger phrase design

**Best Practices**:
- Progressive disclosure (start simple, reveal complexity as needed)
- Bundled resources (templates, examples, schemas)
- Agent-agnostic vs. agent-specific skills
- Validation and quality checks

**Standards Compliance**:
- YAML schema requirements
- Naming conventions (kebab-case)
- Documentation patterns
- Registry updates

## How to Trigger

Say: **"Create a new skill"** or **"Help me build a skill"**

The guide will ask clarifying questions at each phase to ensure your skill is well-designed and standards-compliant.

## What You Get

A complete skill package:
- `skill-name/SKILL.md` (technical implementation)
- `skill-name/README.md` (user-facing documentation)
- Proper frontmatter and directory structure
- Templates and examples as needed
- Validation against Agent Skills Standard

## Key Features

- **Interactive Guidance**: Socratic questioning to refine requirements
- **Standards Enforcement**: Built-in validation checks
- **Examples Library**: Common patterns and anti-patterns
- **Progressive Disclosure**: Start simple, add complexity incrementally
